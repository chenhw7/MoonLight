## 1. 原始需求

用户当前通过填写表单构建简历，修改依赖人工识别。计划进化为AI辅助修改简历，首先需要建立企业JD存储功能。

### 1.1 核心诉求
- 存储企业JD，为后续AI修改简历提供数据基础
- 支持混合录入方式（文本粘贴 + 结构化表单）
- JD独立存储，可与简历建立关联

### 1.2 关键决策点
- 职位描述和任职要求统一为一个字段
- 采用混合录入模式
- JD独立存储架构

## 3. 需求理解

### 3.1 用户场景

**场景1: 快速录入JD**
```
用户在BOSS直聘看到心仪岗位
→ 复制JD文本
→ 粘贴到系统
→ AI自动解析（可选）
→ 保存到JD库
```

**场景2: 结构化录入JD**
```
用户整理目标公司信息
→ 填写公司名、职位名
→ 填写职位描述
→ 添加标签分类
→ 保存到JD库
```

**场景3: 关联简历与JD**
```
用户创建/编辑简历
→ 选择目标JD
→ 系统记录关联关系
→ 为后续AI分析提供上下文
```

### 3.2 核心数据流

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   用户录入   │────▶│   JD存储    │────▶│  简历关联   │
│  (文本/表单) │     │  (独立管理)  │     │ (多对多)   │
└─────────────┘     └──────┬──────┘     └─────────────┘
                           │
                    ┌──────┴──────┐
                    │  AI解析预留  │
                    │ (Phase 2)   │
                    └─────────────┘
```

## 5. 项目特性规范

### 5.1 数据模型规范

```typescript
// JD核心模型
interface JobDescription {
  // 主键
  id: number;
  user_id: number;
  
  // 核心字段（用户要求3项）
  company_name: string;      // 公司名称
  position_name: string;     // 职位名称
  description: string;       // 职位描述（含任职要求）
  
  // 扩展字段（便于筛选管理）
  salary_range?: string;     // 薪资范围
  location?: string;         // 工作地点
  
  // 原始数据
  raw_text?: string;         // 用户粘贴的原始文本
  
  // 元数据
  source?: string;           // 来源平台
  url?: string;              // 原始链接
  tags?: string[];           // 标签数组
  
  // 时间戳
  created_at: Date;
  updated_at: Date;
}
```

### 5.2 关联模型规范

```typescript
// 简历-JD关联
interface ResumeJDLink {
  id: number;
  resume_id: number;         // 简历ID
  jd_id: number;             // JD ID
  notes?: string;            // 用户备注
  created_at: Date;
}
```

### 5.3 页面路由规划

| 路由 | 页面 | 功能 |
|------|------|------|
| `/jds` | JD列表页 | 展示、搜索、筛选、删除 |
| `/jds/new` | 新建JD页 | 录入新JD |
| `/jds/:id` | JD详情页 | 查看、编辑JD |
| `/jds/:id/edit` | 编辑JD页 | 修改JD信息 |
